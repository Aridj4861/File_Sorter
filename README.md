# FileSorter

FileSorter — консольное приложение на C++ для автоматической сортировки файлов по категориям. Программа читает правила из конфигурационного файла, распределяет файлы по папкам, обнаруживает дубликаты по контрольной сумме, обрабатывает конфликты имён, ведёт лог-файл и выводит итоговую статистику.

## Основной функционал

Программа реализует следующий функционал:

* выбор исходной папки для сортировки файлов;
* чтение конфигурационного файла с правилами сортировки;
* автоматическое создание папок назначения по категориям файлов;
* сортировка файлов по расширениям по правилам из конфигурационного файла;
* обработка файлов без расширения;
* обработка файлов с расширениями, которые не указаны в конфигурации;
* проверка конфликта имён при перемещении файлов;
* вычисление контрольной суммы файлов;
* обнаружение дубликатов по контрольной сумме;
* переименование файла, если имя совпадает, но содержимое отличается;
* ведение лог-файла;
* вывод итоговой статистики после завершения работы программы.

## Формат конфигурационного файла

Правила сортировки задаются в текстовом файле. Каждая строка описывает одну категорию файлов:

```text
категория=расширение1,расширение2,расширение3
```

Пример файла `examples/config.txt`:

```text
# Sorting rules
images=jpg,jpeg,png,gif,bmp
documents=txt,pdf,docx,rtf,md
archives=zip,rar,7z
code=cpp,h,hpp,py,js,html,css
```

Слева указывается название папки-категории, справа — расширения файлов через запятую.

Пустые строки и строки, начинающиеся с символа `#`, игнорируются.

## Категории по умолчанию

Если файл не подходит под правила из конфигурационного файла, программа использует служебные категории:

* `other` — файлы с неизвестными расширениями;
* `no_extension` — файлы без расширения;
* `duplicates` — найденные дубликаты.


## Сборка проекта

Проект собирается с помощью CMake.

Команды для сборки из корневой папки проекта:

```bash
cmake -S . -B build
cmake --build build
```

После сборки исполняемый файл будет создан в папке сборки.

## Запуск программы

Общий формат запуска:

```bash
FileSorter.exe --input <folder> --config <file> --log <file>
```

Пример:

```bash
FileSorter.exe --input examples/input --config examples/config.txt --log sorter.log
```

Параметры:

* `--input` — путь к папке с файлами для сортировки;
* `--config` — путь к конфигурационному файлу;
* `--log` — путь к лог-файлу;
* `--help` или `-h` — вывод справки по запуску.

## Демонстрационный запуск

Для удобной демонстрации предусмотрен файл:

```text
run_demo.bat
```

Он выполняет следующие действия:

1. очищает старую демонстрационную папку `examples/input`;
2. копирует исходные демонстрационные файлы из `examples/demo_source`;
3. запускает программу FileSorter;
4. оставляет результат сортировки в `examples/input`;
5. создаёт лог-файл `sorter.log`.

Для запуска демонстрации достаточно дважды нажать на `run_demo.bat`.

Исходные демонстрационные файлы хранятся в папке:

```text
examples/demo_source
```

Эта папка не изменяется программой и используется как эталон для повторных запусков.

## Очистка результатов демонстрации

Для удаления результатов демонстрационного запуска используется файл:

```text
del_results.bat
```

Он удаляет:

* папку `examples/input`;
* файл `sorter.log`.

Папка `examples/demo_source` при этом не удаляется.

## Запуск тестов

Тесты написаны с использованием библиотеки doctest и запускаются через `ctest`.

После сборки проекта можно выполнить:

```bash
cd build
ctest --output-on-failure
```

Также для удобства есть файл:

```text
run_tests.bat
```

Он запускает тесты из папки сборки Visual Studio/CMake.

## Что проверяют тесты

Тесты проверяют:

* нормализацию расширений файлов;
* разбор строк со списком расширений;
* генерацию свободного имени при конфликте;
* чтение корректного конфигурационного файла;
* обработку некорректного конфигурационного файла;
* вычисление контрольной суммы;
* определение одинаковых и разных файлов по контрольной сумме;
* интеграционную сортировку файлов по категориям.

## Контрольная сумма

Для обнаружения дубликатов используется 64-битная контрольная сумма FNV-1a.

Она применяется для практического сравнения содержимого файлов. Алгоритм не предназначен для криптографической защиты и используется только для определения одинаковых файлов в рамках работы программы.

## Обработка ошибок

Ошибки в программе обрабатываются через исключения. Например, исключения выбрасываются при следующих ситуациях:

* не указан обязательный аргумент командной строки;
* не удалось открыть конфигурационный файл;
* конфигурационный файл содержит некорректную строку;
* исходная папка не существует;
* исходный путь не является папкой;
* не удалось открыть файл для вычисления контрольной суммы;
* не удалось открыть или записать лог-файл.

В функции `main` исключения перехватываются через `try/catch`, после чего пользователю выводится сообщение об ошибке и пример запуска программы.

## Основные модули проекта

### `Config`

Отвечает за чтение конфигурационного файла и определение категории по расширению файла.

### `PathUtils`

Содержит вспомогательные функции для работы со строками, расширениями и путями файлов.

### `Checksum`

Вычисляет контрольную сумму файла и преобразует её в шестнадцатеричную строку.

### `Logger`

Записывает действия программы в лог-файл.

### `Statistics`

Хранит и выводит итоговую статистику сортировки.

### `FileSorter`

Основной модуль сортировки. Он проходит по файлам исходной папки, определяет категории, создаёт папки назначения, проверяет дубликаты, решает конфликты имён, перемещает файлы и обновляет статистику.

## Используемые технологии

* C++20;
* CMake;
* стандартная библиотека C++;
* `std::filesystem` для работы с файлами и папками;
* doctest для тестирования;
* clang-format для форматирования кода.

## Примечания

В проекте не используется прямое управление памятью через `new`, `delete`, `malloc`, `free` и подобные функции. Для хранения данных используются контейнеры стандартной библиотеки C++.

Все пути задаются пользователем при запуске программы. В примерах используются относительные пути и прямые слеши.
